# PR Template Draft Generator

作業した内容からプロジェクト標準に準拠したPRテンプレートの草案を生成する。

## 手順

### Step 1: 変更内容の分析

以下のコマンドを実行して変更内容を把握する:
1. `git status` - 変更されたファイルを確認
2. `git diff --stat` - 変更の統計を確認
3. `git diff` - 実際の変更内容を確認
4. `git log -5 --oneline` - 最近のコミット履歴を確認

### Step 2: PRテンプレートの確認

`.github/pull_request_template.md` を読んで、テンプレート形式を確認する。

### Step 3: PR草案の生成

**重要**: 以下の点を守ること:
- 全ての内容は**日本語**で記述する
- テンプレートの形式に厳密に従う
- 変更内容を具体的かつ簡潔に記述する
- 変更の種類は該当するものにチェックを入れる

### Step 4: 出力形式

**必ず単一のMarkdownコードブロック内にPR本文を出力する**ことで、ユーザーがそのままコピー&ペーストできるようにする。

出力例:
```markdown
## 概要
[PRの目的を簡潔に説明]

## 変更内容
- [変更点1]
- [変更点2]

## 変更の種類
- [x] 該当する種類

## 関連Issue
Closes #[番号]

## テスト方法
1. [テスト手順]

## スクリーンショット/動画
[必要に応じて]

## チェックリスト
- [ ] コードが本プロジェクトのコーディング規約に従っている
- [ ] 自分のコードをセルフレビューした
- [ ] コードに必要なコメントを追加した
- [ ] **ドキュメントを確認・更新した**
- [ ] 新しい警告が発生していない
- [ ] テストを追加した（必要な場合）
- [ ] すべてのテストがパスした
- [ ] ビルドが成功している

## その他
[レビュアーへの補足]
```

## 注意事項

- コミットやプッシュは行わない
- 不明な情報（Issue番号、テスト方法など）は確認を求める
- ブランチ命名規則（`feature/{PR番号}`）を確認し、番号を関連Issueに使用する
